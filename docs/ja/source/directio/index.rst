===============================
データの直接入出力 - Direct I/O
===============================
Direct I/OはHadoopクラスターからバッチの入出力データを直接読み書きするための機構です。

ThunderGateやWindGateと異なりデータ転送用の特別なツールは不要ですが、代わりにHadoopクラスターから直接参照できないリソースを利用できません。
主に転送に時間のかかるデータを入出力する場合に、Direct I/Oは適しています。

主に以下のような特徴があります。

高速な入出力
============
入出力データの余計な転送を行わず、Hadoopクラスターから直接データを分散して読み書きするため、高速な入出力を行えます。

柔軟なデータソースの設定
========================
バッチアプリケーションの開発と、運用時に利用するデータソースや入出力位置の指定を分離して行えます。
また、データソースや入出力位置は複数を組み合わせて利用可能です。

柔軟な入出力ファイルパスの指定
==============================
入力パスにはバッチ引数やワイルドカード、サブディレクトリ検索などを指定できます。
出力パスにはバッチ引数のほかデータの内容などを指定でき、ファイルの内容も特定のプロパティでソートできます。
